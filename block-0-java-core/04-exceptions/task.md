# Задание 4 — Исключения

**Цель:** Написать код с валидацией, кастомным исключением и обработкой ошибок.

---

## Что нужно сделать

Расширь класс `Product` из задания 2 (или создай новый).

### Шаг 1 — Кастомное исключение

Создай класс `InvalidProductException` (наследник `RuntimeException`).
Конструктор принимает сообщение об ошибке.

### Шаг 2 — Валидация в классе Product

Добавь сеттеры с проверками — бросать `InvalidProductException` если:
- `name` — пустая строка или null
- `price` — меньше нуля
- `stock` — меньше нуля

### Шаг 3 — Обработка

В `main` попробуй создать продукт с некорректными данными.
Обработай исключение через `try/catch` и выведи понятное сообщение:
```
Ошибка создания продукта: Цена не может быть отрицательной
```

### Шаг 4 — Несколько сценариев

Проверь все три некорректных случая (три отдельных `try/catch` блока или один с разными данными).

---

## Задание выполнено, если:

- [ ] Создан кастомный класс исключения `InvalidProductException`
- [ ] Валидация работает для всех трёх полей
- [ ] Исключение поймано в `catch` и выведено читаемое сообщение
- [ ] Программа не падает с необработанным исключением
- [ ] Корректные данные создают продукт без ошибок

---

## Подсказка

`throw new InvalidProductException("текст")` — бросить исключение.
Проверка на null и пустую строку: `name == null || name.isEmpty()`.
