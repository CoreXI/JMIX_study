## Зачем это нужно

В Node.js ты пишешь `let x = 5` — и JS сам разбирается какой это тип.
В Java компилятор должен знать тип **до запуска программы**. Это называется **статическая типизация**.

Зачем? Ошибки типов ловятся на этапе компиляции, а не в проде в 3 ночи.

---

## Примитивные типы

Java имеет 8 встроенных типов:

| Тип | Размер | Что хранит | Диапазон / примечание |
|-----|--------|-----------|----------------------|
| `byte` | 8 бит | целое число | −128 .. 127 |
| `short` | 16 бит | целое число | −32 768 .. 32 767 |
| `int` | 32 бита | целое число | −2 147 483 648 .. 2 147 483 647 |
| `long` | 64 бита | большое целое | ±9 квинтиллионов; литерал: `100L` |
| `float` | 32 бита | число с плавающей точкой | ~7 знаков после запятой; литерал: `3.14f` |
| `double` | 64 бита | число с плавающей точкой | ~15 знаков после запятой; используется по умолчанию |
| `boolean` | — | логический | `true` / `false` |
| `char` | 16 бит | один Unicode-символ | `'A'`, `'€'`; диапазон 0..65 535 |

На практике 90% времени используются `int`, `long`, `double`, `boolean` и `char`.
`byte` и `short` — для экономии памяти в массивах; `float` — когда точность `double` избыточна (игры, графика).

```java
int age = 25;
double price = 9.99;
boolean isActive = true;
char grade = 'A';
```

> Примитивы — не объекты. У них нет методов. `int` это просто число в памяти.

## String — особый случай

`String` — это класс, не примитив. Но используется как примитив:

```java
String name = "Alice";
String greeting = "Hello, " + name; // конкатенация через +
```

Важно: строки в Java **неизменяемые** (immutable). Каждая операция создаёт новую строку.

## var — вывод типа (Java 10+)

Если тип очевиден из правой части — можно написать `var`:

```java
var age = 25;        // компилятор знает: это int
var name = "Alice";  // компилятор знает: это String
```

`var` — это не `let` из JS. Тип всё равно фиксируется при объявлении, просто ты его не пишешь руками.

## Объявление переменных

```java
// Объявление с присвоением
int x = 10;

// Объявление без присвоения (нужно присвоить до использования)
int y;
y = 20;

// final — константа, нельзя переприсвоить (аналог const в JS)
final int MAX = 100;
```

## Классы-обёртки (Wrapper classes)

У каждого примитива есть класс-обёртка — нужны там, где требуется объект (коллекции, дженерики):

| Примитив | Обёртка |
|----------|---------|
| `int` | `Integer` |
| `double` | `Double` |
| `boolean` | `Boolean` |
| `long` | `Long` |

```java
Integer boxed = 42;       // автоупаковка (autoboxing)
int unboxed = boxed;      // автораспаковка (unboxing)
```

Java делает это автоматически, но знать надо — иногда это источник багов (NPE при unboxing null).

---

## Приведение типов

```java
int a = 10;
double b = a;       // неявное: int → double (безопасно, без потери данных)

double x = 9.99;
int y = (int) x;    // явное приведение: double → int = 9 (дробная часть обрезается)
```

## Суть

> Java знает типы всех переменных **до запуска**. Это делает код предсказуемым и ловит целый класс ошибок на этапе компиляции. Цена — чуть больше явности при написании.
