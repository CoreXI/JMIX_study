# Задание 6 — Stream API

**Цель:** Решить аналитическую задачу только через Stream API — без циклов `for`/`while`.

---

## Контекст

Есть список сотрудников. Каждый сотрудник — объект с полями:
- `name` (String)
- `department` (String) — отдел: `"IT"`, `"HR"`, `"Finance"`
- `salary` (double)
- `yearsOfExperience` (int)

Создай класс `Employee` с этими полями и конструктором.

---

## Что нужно сделать

Создай список из 8-10 сотрудников с разными отделами и зарплатами.

Реши следующие задачи **только через Stream API** (без for-циклов):

### 1. Фильтрация
Выведи имена всех сотрудников из отдела `"IT"`.

### 2. Трансформация
Создай `List<String>` в формате `"Имя — Отдел"` для всех сотрудников.

### 3. Агрегация
Найди среднюю зарплату по всей компании.

### 4. Группировка
Создай `Map<String, List<Employee>>` — сотрудники, сгруппированные по отделу.
Выведи для каждого отдела количество сотрудников.

### 5. Сложная выборка
Найди самого высокооплачиваемого сотрудника среди тех, у кого опыт больше 3 лет.
Используй `filter` + `max` + `Optional`.

### 6. Строка-сводка
Выведи через запятую имена всех сотрудников с зарплатой выше средней.

---

## Задание выполнено, если:

- [ ] Ни одного `for`/`while` цикла — только Stream API
- [ ] Использованы: `filter`, `map`, `collect`, `groupingBy`, `max`, `average`, `joining`
- [ ] `Optional` обработан корректно (`.orElse()` или `.ifPresent()`, не `.get()` без проверки)
- [ ] Задача 6 реально вычисляет среднюю — не захардкожена
- [ ] Код компилируется и выдаёт корректные результаты

---

## Подсказка

Задача 6 — в два прохода: сначала вычисли среднюю через `.average()`, потом фильтруй и соединяй. Или реши через `Collectors.partitioningBy`.
