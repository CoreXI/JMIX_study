# Задание 2 — Классы и ООП

**Цель:** Создать иерархию классов, применить инкапсуляцию, наследование и интерфейс.

---

## Что нужно сделать

Смоделируй систему сотрудников компании.

### Шаг 1 — Интерфейс

Создай интерфейс `Payable` с методом:
```java
double calculateSalary();
```

### Шаг 2 — Базовый класс

Создай класс `Employee` (сотрудник):
- Поля (все `private`): `name` (String), `id` (int)
- Геттеры для обоих полей
- Конструктор с параметрами `name` и `id`
- Метод `getInfo()` — возвращает строку `"[id] name"`

### Шаг 3 — Наследники

Создай два класса, которые наследуют `Employee` и реализуют `Payable`:

**`FullTimeEmployee`** — штатный сотрудник:
- Дополнительное поле `monthlySalary` (double)
- `calculateSalary()` возвращает `monthlySalary`

**`ContractEmployee`** — контрактник:
- Дополнительные поля: `hourlyRate` (double), `hoursWorked` (int)
- `calculateSalary()` возвращает `hourlyRate * hoursWorked`

### Шаг 4 — Запуск

В `main` создай по одному объекту каждого типа и выведи:
```
[1] Алиса | Зарплата: 120000.0
[2] Боб | Зарплата: 75000.0
```

---

## Задание выполнено, если:

- [ ] Интерфейс `Payable` создан и реализован обоими классами
- [ ] Поля `Employee` объявлены как `private`, доступ через геттеры
- [ ] `@Override` стоит над переопределёнными методами
- [ ] Конструктор наследника вызывает `super(...)`
- [ ] Программа компилируется и выводит корректный результат

---

## Подсказка

Переменную типа `Payable` можно использовать для хранения любого объекта, который реализует этот интерфейс — попробуй создать массив `Payable[]` и пройтись по нему циклом.
