## Зачем это нужно

Прежде чем писать код — нужно понять среду, в которой ты работаешь.
IntelliJ IDEA — стандарт для Java-разработки. Большинство команд используют именно её.

---

## Интерфейс: где что находится

```
┌─────────────────────────────────────────────────────┐
│  Меню: File  Edit  View  Run  Git  Help             │  ← верхняя панель
├──────────┬──────────────────────────┬───────────────┤
│          │                          │               │
│ Project  │      Редактор кода       │  Structure /  │
│ (дерево  │                          │  другие       │
│  файлов) │                          │  панели       │
│          │                          │               │
├──────────┴──────────────────────────┴───────────────┤
│  Run / Terminal / Problems / Git                    │  ← нижняя панель
└─────────────────────────────────────────────────────┘
```

### Левая панель — Project
Дерево файлов проекта. Открывается через `View → Tool Windows → Project` или `Alt+1`.

Главное что видишь:
- `src/` — исходный код
- `pom.xml` / `build.gradle` — конфигурация проекта (зависимости, настройки сборки)

### Центр — Редактор
Здесь пишешь код. Вкладки вверху — открытые файлы.

### Нижняя панель
| Вкладка | Что делает |
|---------|-----------|
| **Run** | Вывод программы при запуске |
| **Terminal** | Встроенный терминал (как обычный bash/zsh) |
| **Problems** | Ошибки и предупреждения в коде |
| **Git** | История изменений, diff |

## Кнопки запуска — главные три

```
▶  Run       — запустить программу
🐛  Debug     — запустить с отладчиком (можно ставить точки остановки)
🔨  Build     — скомпилировать без запуска
```

Находятся в правом верхнем углу. Также через меню `Run`.

Быстрее: кликнуть зелёный треугольник рядом с методом `main()` прямо в редакторе.

## Горячие клавиши — минимум для старта

| Действие | Mac | Windows/Linux |
|----------|-----|---------------|
| Запустить программу | `Ctrl+R` | `Shift+F10` |
| Найти файл по имени | `Shift+Shift` (двойной) | `Shift+Shift` |
| Найти текст в проекте | `Cmd+Shift+F` | `Ctrl+Shift+F` |
| Переименовать | `Shift+F6` | `Shift+F6` |
| Автодополнение | `Ctrl+Space` | `Ctrl+Space` |
| Показать подсказку по ошибке | `Alt+Enter` | `Alt+Enter` |
| Форматировать код | `Cmd+Alt+L` | `Ctrl+Alt+L` |
| Закомментировать строку | `Cmd+/` | `Ctrl+/` |

`Alt+Enter` — твой лучший друг. Красная ошибка? Ставишь курсор, жмёшь `Alt+Enter` — IDEA предложит варианты исправления.

## Что уже есть "из коробки"

IntelliJ IDEA Community сразу умеет:

- **Подсветка синтаксиса** для Java, Kotlin, XML, JSON, Markdown
- **Автодополнение** — начинаешь писать, IDE предлагает варианты
- **Инспекции** — IDE подчёркивает подозрительный код ещё до запуска
- **Рефакторинг** — переименовать класс / метод во всём проекте сразу
- **Отладчик** — пошаговое выполнение, просмотр переменных
- **Git-интеграция** — commit, push, diff прямо из IDE
- **Встроенный терминал** — не нужно переключаться в отдельное окно
- **Запуск тестов** — JUnit запускается одной кнопкой

Дополнительно устанавливать плагины для базовой Java-разработки **не нужно**.

## Создание первого проекта

`File → New Project → Java`

Выбрать:
- **Build system:** IntelliJ (для простых учебных проектов) или Maven (для всего остального)
- **JDK:** выбрать установленный (если нет — IDEA предложит скачать)

После создания появится `src/Main.java` с готовым `main()` — можно сразу запускать.

---

## JDK — что это и где взять

JDK (Java Development Kit) — компилятор + runtime для Java. Без него код не запустится.

Если не установлен: при создании проекта IDEA предлагает скачать сама — выбирай **Amazon Corretto 21** или **Eclipse Temurin 21** (LTS-версия, стабильная).
