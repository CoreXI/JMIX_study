# Задание 5 — Generics

**Цель:** Написать свой обобщённый класс и метод, понять зачем это нужно.

---

## Что нужно сделать

### Шаг 1 — Обобщённый контейнер результата

Создай класс `Result<T>` — обёртка над результатом операции (успех или ошибка).
Аналог `{ data, error }` из JS или `Either` из функционального программирования.

Поля:
- `T value` — данные (если успех)
- `String errorMessage` — сообщение об ошибке (если ошибка)
- `boolean success` — флаг успеха

Методы:
- `static <T> Result<T> ok(T value)` — фабричный метод для успеха
- `static <T> Result<T> fail(String message)` — фабричный метод для ошибки
- `getValue()`, `getErrorMessage()`, `isSuccess()`

### Шаг 2 — Используй Result

Напиши метод `findProduct(List<String> products, String name)`:
- Если продукт найден — возвращает `Result<String>` с успехом
- Если не найден — возвращает `Result<String>` с ошибкой `"Продукт не найден: [name]"`

### Шаг 3 — Обобщённый метод поиска максимума

Напиши статический метод `max(List<T> list)` где `T extends Comparable<T>`.
Метод возвращает `Optional<T>` — максимальный элемент или пустой Optional если список пуст.

Проверь на `List<Integer>` и `List<String>`.

---

## Задание выполнено, если:

- [ ] `Result<T>` работает с любым типом (проверь с `String` и `Integer`)
- [ ] Фабричные методы `ok()` и `fail()` объявлены как `static <T>`
- [ ] Метод `max()` использует ограничение `extends Comparable<T>`
- [ ] `Optional` использован корректно (не `null` в качестве "нет значения")
- [ ] Код компилируется и работает

---

## Подсказка

`Comparable<T>` — встроенный интерфейс с методом `compareTo()`. Строки и числа его реализуют.
`compareTo` возвращает: отрицательное — если меньше, 0 — если равно, положительное — если больше.
