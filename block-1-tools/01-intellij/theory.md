## Зачем это нужно

IntelliJ IDEA — это среда разработки (IDE), в которой ты будешь писать весь Java/JMIX-код. Она заменяет текстовый редактор и добавляет: автодополнение, рефакторинг, отладчик, интеграцию с Git и Gradle.

Аналог из Node.js-мира: VS Code, но заточенный под Java с гораздо более глубоким пониманием кода.

---

## Структура проекта

Когда открываешь Java-проект в IntelliJ, видишь:

```
project/
├── src/
│   └── main/
│       └── java/        ← здесь твой код
│           └── com/example/
└── build.gradle         ← конфигурация сборки
```

- **Project** (левая панель) — дерево файлов
- **Editor** (центр) — код
- **Run/Debug** (внизу) — вывод программы

## Отладчик (Debugger)

Отладчик позволяет остановить выполнение программы в любом месте и посмотреть значения переменных.

**Как использовать:**
1. Кликни на номер строки — появится красная точка (**breakpoint**)
2. Запусти программу в режиме отладки (`Ctrl+D` / `Shift+F9`)
3. Программа остановится на breakpoint
4. Смотри переменные в панели **Variables** внизу

**Кнопки управления:**
- **Step Over** (`F8`) — выполнить текущую строку, перейти к следующей
- **Step Into** (`F7`) — зайти внутрь вызываемого метода
- **Resume** (`F9`) — продолжить до следующего breakpoint

## Live Templates (сниппеты)

IntelliJ умеет разворачивать короткие аббревиатуры в код:

| Набери | Получишь |
|--------|----------|
| `sout` + Tab | `System.out.println()` |
| `fori` + Tab | цикл `for (int i = 0; ...)` |
| `iter` + Tab | цикл `for (T item : collection)` |
| `psvm` + Tab | `public static void main(String[] args)` |

## Суть в том, что...

IntelliJ — это не просто редактор, а инструмент, который понимает твой код. Чем лучше ты знаешь горячие клавиши, тем быстрее пишешь. Отладчик важнее `System.out.println` — привыкай его использовать с первых дней.
